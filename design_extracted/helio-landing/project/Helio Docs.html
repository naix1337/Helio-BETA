Erste Schritte Dashboard installieren

Dashboard installieren

Bringe das Helio-Dashboard in unter einer Minute auf deinem eigenen Server zum Laufen — ein einzelnes Binary, keine externe Datenbank, volle Datenhoheit.

Voraussetzungen

Bevor du startest, stelle sicher, dass dein Host die folgenden Anforderungen erfüllt. Helio ist bewusst schlank gehalten und läuft auch auf einem Raspberry Pi.

  • Linux-Host — x86-64 oder ARM, Kernel ≥ 4.4 (z. B. Debian, Ubuntu, Alpine).
  • Docker ≥ 20.10 — optional, aber empfohlen für die Container-Erkennung.
  • 512 MB freier RAM — Helio selbst belegt im Schnitt nur rund 42 MB.
  • Offener Port 9300 — für das Dashboard und die Agent-Verbindungen.

Installation

Der schnellste Weg ist das offizielle Installationsskript. Es erkennt deine Plattform automatisch, lädt das passende Binary herunter und richtet einen systemd-Dienst ein.

bash 
$ curl -fsSL https://helio.sh/install | sh
Tipp

Du kannst das Skript vor der Ausführung prüfen: Öffne https://helio.sh/install einfach im Browser. Helio installiert nichts außerhalb von /opt/helio.

Alternative: Docker

Wenn du lieber containerisiert bleibst, startet der folgende Befehl Helio inklusive automatischer Container-Erkennung über den Docker-Socket.

docker 
$ docker run -d --name helio \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -p 9300:9300 \
    helio/helio:latest
Warnung

Das Einhängen des Docker-Sockets gibt dem Container weitreichende Rechte auf dem Host. Setze Helio in produktiven Umgebungen hinter einen Reverse-Proxy mit TLS und schränke den Zugriff auf Port 9300 per Firewall ein.

Erster Start

Nach der Installation ist das Dashboard sofort erreichbar. Folge diesen Schritten, um deinen ersten Node hinzuzufügen.

Dashboard öffnen

Rufe http://localhost:9300 auf. Beim ersten Aufruf wirst du durch die Einrichtung des Admin-Kontos geführt.

Admin-Konto anlegen

Lege Benutzernamen und ein sicheres Passwort fest. Es werden keinerlei Daten an externe Dienste übertragen.

Ersten Agent verbinden

Kopiere den angezeigten Verbindungsbefehl auf einen weiteren Host. Der Agent verbindet sich verschlüsselt mit deiner Instanz und erscheint binnen Sekunden im Dashboard.

Hinweis

Du kannst weitere Nodes jederzeit unter Nodes & Agents hinzufügen. Eine zentrale Instanz verwaltet problemlos mehrere hundert Agents.
